## 2. API 前缀规则

### 2.1 Admin 管理接口

| 项目 | 规则 |
|---|---|
| Controller 内部前缀 | `/delta/**` |
| 网关完整外部前缀 | `/admin-api/delta/**` |
| Vue3 Axios 请求路径 | `/delta/**` |
| 登录身份 | Admin 用户 |
| 常见权限方式 | `@PreAuthorize`、权限编码 |

示例：

```text
Controller：
@RequestMapping("/delta/service-order")

完整外部 URL：
/admin-api/delta/service-order/page

Vue3 请求路径：
/delta/service-order/page
```

Vue3 前端代码不得直接写 `/admin-api/delta/**`，避免 Axios 重复拼接。

### 2.2 App / UniApp 接口

| 项目 | 规则 |
|---|---|
| Controller 内部前缀 | `/delta/**` |
| 网关完整外部前缀 | `/app-api/delta/**` |
| UniApp 请求路径 | `/delta/**` |
| 登录身份 | Member 用户 |
| 当前会员来源 | `getLoginUserId()` |

示例：

```text
Controller：
@RequestMapping("/delta/worker-order")

完整外部 URL：
/app-api/delta/worker-order/page

UniApp 请求路径：
/delta/worker-order/page
```

UniApp 代码不得在业务 API 路径中重复写 `/app-api/delta/**`。

### 2.3 Controller 路径限制

Java Controller 的 `@RequestMapping` 中不得重复声明：

```text
/admin-api
/app-api
```

统一只声明 `/delta/**`，外部前缀由网关和服务配置处理。

### 2.4 身份和租户通用规则

Admin 接口的操作身份来自当前后台登录用户，不得由请求参数决定 `operatorId`、`adminUserId` 或 `tenantId`。

App 接口的会员身份来自：

```java
getLoginUserId()
```

不得由请求参数决定：

```text
memberUserId
ownerMemberId
tenantId
clubId
claimedClubTenantId
sourceTenantId
workerTenantId
operatorId
operatorType
```

打手身份必须通过当前登录会员解析真实 `delta_worker` 档案，不得只信任客户端提交的 `workerId`。

俱乐部老板身份必须通过以下可信关系解析：

```text
当前登录 memberUserId
→ delta_club_profile.owner_member_id
→ 可信俱乐部档案
```

### 2.5 数据租户通用规则

普通同租户订单：

```text
服务订单租户 = 打手租户
```

俱乐部跨租户订单：

```text
服务订单租户 = sourceTenantId
打手和俱乐部租户 = claimedClubTenantId
```

服务履约业务数据原则上写入服务订单来源租户：

```text
服务订单
派单记录
进度
凭证
订单日志
验收
返工
结算
结算日志
买家 Outbox
```

打手资料、打手工作状态和打手 Outbox 写入俱乐部租户。

忽略租户过滤只允许用于必要的可信定位查询，不允许包裹完整业务写流程。
